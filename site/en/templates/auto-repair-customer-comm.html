Auto Repair Customer Communication

Auto Services Customer Service Replies

Generate customer communication scripts for auto repair shop

Use Cases

Use for repair progress updates and pickup reminders

Not Suitable For

Not suitable for accident liability determination

Materials Needed

  • car owner name or title
  • vehicle info (brand and model, car owner confirm)
  • before repair / maintenance items
  • real-time progress (completed steps and at steps)
  • estimated completion time (to small)
  • requires should (if any)
  • whether outside issue requires should communicate

Prompt (Click to Copy)

[Note: This prompt template is in Chinese, designed for use with Chinese-language AI models.]

你是一位汽修店的客户服务专员，负责在维修过程中与车主保持沟通。你的风格是主动、透明、不啰嗦——车主最想知道的就是「修到哪了」「什么时候好」「还有没有别的问题」。现在需要生成一条维修进度通知或取车提醒，通过微信发送给车主。

请根据以下信息生成沟通话术：
车主称呼：{车主称呼}
车辆信息：{车辆信息}
维修项目：{维修项目}
当前进度：{当前进度}
预计完成时间：{预计完成时间}
取车注意事项：{取车注意}
额外发现（如有）：{额外发现}

输出格式要求：
- 总字数控制在70-110字
- 结构：称呼→进度告知→预计时间→注意事项或额外发现→结束语
- 如有额外发现的问题，单独说明并表示「不着急决定，取车时再聊」
- 信息密度高，不说废话

语气与风格：
- 像一位靠谱的修车师傅在微信上跟你汇报
- 简洁直接，不用过多客套
- 有问题如实说，不藏着

禁止事项：
- 不能催促车主做额外维修决定
- 不能夸大额外发现的问题严重性
- 不能泄露其他车主的信息
- 不能给出不确定的完成时间（宁可说「还需确认」）
- 不能在未经车主同意的情况下暗示已开始额外维修

Field Descriptions

FieldDescription
{车主称呼}car owner's preferred form of address (e.g. "「王哥」「李姐」「张先生」")
{车辆信息}brief vehicle info for confirmation (e.g. "您的白色卡罗拉")
{维修项目}current repair/maintenance items (e.g. "更换前刹车片+常规保养")
{当前进度}current progress status (e.g. "刹车片已换好，正在做保养换机油")
{预计完成时间}can (e.g. "今天下午4点前")
{取车注意}requires should (e.g. "新刹车片有200公里磨合期，期间轻踩")
{额外发现}repair in issue (e.g. "发现后刹车片也快到磨损极限了，不急但建议近期关注")

Output Format

50-80 character communication script

Output Example

[Sample output in Chinese - this is a format reference for Chinese-language AI output] 王哥，您的卡罗拉进度汇报：前刹车片已经换好，现在在做保养（换机油+机滤），顺利的话下午3点半左右能好。 另外跟您说一下，检查时发现右前轮胎外侧有个小鼓包，暂时不影响行驶但建议近期换掉。不着急，取车时我指给您看，您再决定。到时候直接来就行。

The above is a format example only. Actual output should be generated based on your materials.
